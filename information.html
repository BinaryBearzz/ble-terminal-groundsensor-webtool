2) Parser Rules

  • รูปแบบ command ใช้แนว AT+CMD=param1,param2,...
  • ระบบแยกชื่อคำสั่งด้วยเครื่องหมาย =
  • ถ้า command มี \r ท้ายบรรทัด จะถูกตัดทิ้งด้วย trim()
  • unknown command จะ ไม่มี response โดย default เพราะ processCommand() คืนค่าว่าง
  • response บางคำสั่งเป็นแบบ multi-line เพราะ callback ส่งออก UART โดยตรงก่อนคืนค่าปิดท้าย 
Host line ending recommendation: \r\n
Examples:
AT

AT+SEND=1,123,65,66,67

AT+SYNCTIME=1783061906

3) Command Size / Payload Limits

Topic Limit Notes
UART command line max size Not explicitly bounded in parser อ่านเข้า String ผ่าน readStringUntil('\n'); code ไม่ได้ตั้ง max line length โดยตรง
LoRa raw buffer BUFFER_SIZE = 51 bytes buffer ภายใน packet
LoRa application max packet target MAX_BUFFER_SIZE = 30 bytes ใช้ประกอบข้อมูล info
AT+SEND data bytes 21 bytes max เกินแล้วตอบ ERROR,DATA:>21
Package Format sizeOfAll[0],unix[1-4],deviceId[5-6],projectId[7],data(max21)[8-n(<=28)],crc[sizeOfAll-1]

หมายเหตุ: “max size command” ในระดับ UART line ไม่มี hard limit ชัดเจนใน parser แต่คำสั่งที่ส่งข้อมูลจะถูกจำกัดด้วยจำนวน data byte ตามตารางด้านบน

4) RX Command Specification (Host -> Device)

Command Syntax Description Immediate / Final Response Validation / Notes
AT AT Ping device OK basic connectivity test
AT+SEND AT+SEND=<ch>,<projectId>,<b1>,...,<bn> ส่ง Postman packet 1 ครั้ง ก่อนส่งมี OK:INITSEND,S:<n>,CH:<ch>,PROJ:<project>
จบด้วย OK,SEND:<n>,CH:<ch>,FINISH 		payload max 21 bytes; ถ้า ch เป็น 0 หรือ >8 จะสุ่ม CH 1-8
AT+POST AT+POST=<ch>,<repeat>,<intervalMs>,<projectId>,<b1>,...,<bn> ส่ง Postman แบบวนซ้ำตามช่วงเวลา เริ่มด้วย OK:INITSEND,R:<repeat>,I:<interval>,S:<n>,CH:<ch>,PROJ:<project>
ระหว่างทางมี POSTLOOP:x/y และ OK,SEND:<n>,CH:<ch>,CNT:x/y
จบด้วย OK,SEND:<n>,CH:<ch>,FINISH:<repeat> 		intervalMs ต่ำกว่า 2000 จะถูกบังคับเป็น 2000; ch = -1 ทำ random channel ทุก loop
AT+RST AT+RST รีสตาร์ตบอร์ด OK แล้ว restart ไม่มี final response หลัง reboot
AT+SYNCTIME AT+SYNCTIME=<unixTime> sync เวลาและเปิด flag ใช้เวลาจาก command OK,SYNCTIME:<unixTime>(10 digits) ถ้า valid จะ set rtcOnBoard on run time
AT+RSTTIME AT+RSTTIME reset mock/manual time OK,RSTTIME รีเซ็ตเวลา temp กลับค่า default mock time 1/1/1999
AT+SETDID (Optional) AT+SETDID=<thingId> ตั้ง device / thing ID OK,SETDID:<thingId> ช่วง valid คือ 1..65535; นอกช่วงตอบ ERROR,INVALID DID

5) TX Messages / Status Messages (Device -> Host)

Type Message Pattern When Source
Boot info BLE_NAME:<device_name> ช่วง setup() หลัง init BLE main.cpp
Send init OK:INITSEND,S:<n>,CH:<ch>,PROJ:<project> ก่อน AT+SEND ส่งจริง CommandHandler.cpp
Loop init OK:INITSEND,R:<repeat>,I:<interval>,S:<n>,CH:<ch>,PROJ:<project> เริ่ม AT+POST CommandHandler.cpp
Loop progress POSTLOOP:<count>/<repeat> ทุกครั้งที่ส่งใน loop mode Postman.cpp
Loop send result OK,SEND:<n>,CH:<ch>,CNT:<count>/<repeat> ทุกครั้งที่ส่งใน loop mode Postman.cpp
Single send final OK,SEND:<n>,CH:<ch>,FINISH จบ AT+SEND Postman.cpp
Loop send final OK,SEND:<n>,CH:<ch>,FINISH:<repeat> จบ AT+POST Postman.cpp
LoRa callback success OK,TX หรือ OK,TX:<count> LoRa TxDone callback Postman.cpp
LoRa callback error ERROR,TX หรือ ERROR,TX:<count> LoRa TxTimeout callback Postman.cpp

6) Common Error Responses

Error Meaning
ERROR:CH,... channel parameter ขาดหรือไม่ถูกต้อง
ERROR:PROJ,... / ERROR,P:0 project ID ขาดหรือยังไม่ถูกตั้ง
ERROR:T:0 thing ID ยังไม่ถูกตั้ง
ERROR,DATA:>N จำนวน data byte เกิน limit ของ command/version
ERROR,P_SIZE>N payload เกิน packet limit ภายใน Postman/CHEF
ERROR,INVALID TIME unix time ต่ำกว่า threshold ที่โค้ดยอมรับ
ERROR,INVALID DID thingId นอกช่วง 1..65535
ERROR,INVALID:VER version ของ AT+SATE ไม่ใช่ V1/V2/V3/V4
